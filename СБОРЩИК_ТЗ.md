# Интерфейс сборщика — техническое задание

## Общий сценарий работы

1. Сборщик заходит под своим логином (`sborshik/1234`)
2. На главной странице — кнопка **«Начать сборку»**
3. Выбирает номер двигателя из списка (например: ТВ3-117 №А-2024-001)
4. Открывается рабочий экран сборки

---

## Рабочий экран сборки — структура

### Левая панель (1/4 экрана) — прогресс операций
- Вертикальный список операций сверху вниз
- Каждая операция — строка с номером и коротким названием
- **Текущая операция** — выделена (яркий цвет, жирный шрифт)
- **Выполненные** — серые с галочкой ✓
- **Недоступные** — затемнены, не кликабельны
- Сверху счётчик: `Операция 3 из 12`
- Сборщик не может перейти к следующей операции не завершив текущую

### Центральная зона (2/4 экрана) — детали текущей операции
- Сетка карточек (2-3 в ряд) — по одной на каждую деталь операции
- **Карточка детали:**
  ```
  ┌─────────────────────────┐
  │  Шайба контровочная     │
  │  арт. 0200045038        │
  │                         │
  │  [ячейка]    [кол-во]  │
  │   С1-П2-Я2      11 шт  │
  └─────────────────────────┘
  ```
  - Название детали — крупно сверху
  - Артикул — мелко под названием
  - Левый нижний угол: **номер ячейки** (где лежит деталь)
  - Правый нижний угол: **количество** которое нужно взять
  - Цвет рамки карточки — по светофору: зелёный/жёлтый/красный в зависимости от остатка
  - Если остаток < нужного количества — карточка красная + предупреждение

### Правая панель (1/4 экрана) — информация о двигателе
- Номер двигателя
- Тип (ТВ2-117 / ТВ3-117)
- Дата начала сборки
- Текущая операция (название)
- Общий прогресс в % (прогресс-бар)

## Кнопки управления (внизу экрана)

```
[← Отменить операцию]    [✓ Операция выполнена →]
```

### «Операция выполнена» (зелёная, справа)
- При нажатии — всплывающее окно подтверждения:
  `«Подтвердить списание 11 шт Шайба контровочная + 3 шт Замок?»`
- После подтверждения:
  - Списываются все детали операции из stock
  - Запись в movements (тип: расход, номер двигателя, операция, сборщик)
  - Переход к следующей операции
  - Если это последняя операция — экран завершения сборки

### «Отменить операцию» (серая, слева)
- Откат последней выполненной операции
- Всплывающее окно: `«Вернуть операцию 2 "Сборка компрессора"? Детали будут возвращены на баланс.»`
- После подтверждения:
  - Возвращает списанные детали обратно в stock
  - Помечает операцию как не выполненную
  - Возвращает на предыдущую операцию
- **Нельзя отменить если сборка завершена**

## База данных — новые таблицы

```sql
-- Этапы сборки (операции) для каждого типа двигателя
CREATE TABLE assembly_stages (
    id INTEGER PRIMARY KEY,
    engine_type TEXT NOT NULL,       -- 'ТВ2-117' или 'ТВ3-117'
    stage_number INTEGER NOT NULL,   -- порядковый номер операции
    stage_name TEXT NOT NULL,        -- название операции
    description TEXT                 -- краткое описание
);

-- Детали необходимые для каждой операции
CREATE TABLE stage_parts (
    id INTEGER PRIMARY KEY,
    stage_id INTEGER REFERENCES assembly_stages(id),
    part_id INTEGER REFERENCES parts(id),
    quantity INTEGER NOT NULL        -- сколько штук нужно на 1 двигатель
);

-- Активные сборки двигателей
CREATE TABLE engine_assemblies (
    id INTEGER PRIMARY KEY,
    engine_number TEXT UNIQUE NOT NULL,  -- номер двигателя
    engine_type TEXT NOT NULL,
    current_stage INTEGER DEFAULT 1,     -- текущая операция
    status TEXT DEFAULT 'В работе',      -- В работе / Завершена / Приостановлена
    assembler TEXT NOT NULL,             -- логин сборщика
    started_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    completed_at DATETIME
);

-- История выполненных операций
CREATE TABLE assembly_history (
    id INTEGER PRIMARY KEY,
    assembly_id INTEGER REFERENCES engine_assemblies(id),
    stage_id INTEGER REFERENCES assembly_stages(id),
    completed_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    assembler TEXT,
    is_cancelled INTEGER DEFAULT 0   -- 1 если операция была отменена
);
```

## Маршруты Flask

```
GET  /assembly                    — страница выбора двигателя
POST /assembly/start              — начать сборку (создать запись в engine_assemblies)
GET  /assembly/<id>               — рабочий экран сборки
POST /assembly/<id>/complete      — завершить текущую операцию + списание
POST /assembly/<id>/cancel        — отменить последнюю операцию + возврат деталей
GET  /assembly/<id>/finish        — экран завершения сборки
```

## UX требования

- Кнопка «Операция выполнена» — минимум 60px высота (работа в перчатках)
- Карточки деталей — минимум 200x150px
- Номер ячейки — крупный шрифт, хорошо читается с расстояния
- Автоблокировка кнопки «Выполнена» если остаток < нужного количества
- После каждого действия — flash-сообщение об успехе/ошибке
- Адаптивность: работает на планшете 10" в горизонтальной ориентации
