# Максимальная ёмкость ячеек и доработка прихода

## Логика максимума ячейки

Максимальная ёмкость ячейки = monthly_plan (плановый месячный расход) из таблицы parts.
Это уже есть в БД — не нужно добавлять новое поле.

Добавить в таблицу stock поле:
```sql
ALTER TABLE stock ADD COLUMN max_quantity INTEGER;
```
Заполнить: max_quantity = monthly_plan детали привязанной к ячейке.

---

## Доработка формы прихода (/prikhod)

### Убрать выбор ячейки
- Поле «Ячейка хранения» убрать из формы полностью
- Ячейка определяется автоматически по выбранной детали (из stock.cell_id)
- Показывать как read-only информацию: «Ячейка: С1-П1-Я1» под названием детали

### Автоматический расчёт количества к приходу
При выборе детали — автоматически вычислять и подставлять:
```
количество к приходу = max_quantity - текущий остаток
```
Например: max = 60, остаток = 2 → подставить 58

- Поле количества остаётся редактируемым — можно уменьшить
- Нельзя ввести больше чем (max_quantity - текущий остаток) — валидация на фронте и бэке
- Под полем показывать подсказку: «Макс. к приходу: 58 шт (до заполнения ячейки)»
- Если остаток уже равен максимуму — показать предупреждение «Ячейка заполнена» и заблокировать кнопку

### Что показывать при выборе детали
Карточка под поиском:
```
Артикул: 0080004096
Наименование: Замок
Ячейка: С1-П1-Я1
Текущий остаток: 2 шт
Максимум ячейки: 60 шт
Свободно: 58 шт  ← подсвечивать зелёным/жёлтым/красным
```

---

## Доработка модального окна заявки на пополнение

То же самое — при открытии модала:
- Автоматически вычислять quantity = max_quantity - текущий остаток
- Показывать: «Рекомендуется: 58 шт (до заполнения ячейки)»
- Нельзя заказать больше чем свободно в ячейке
- Ячейку не показывать и не запрашивать

## Доработка таблицы ячеек (/cells)

Добавить колонку «Заполненность»:
- Прогресс-бар: текущий остаток / максимум × 100%
- Цвет прогресс-бара: зелёный >60%, жёлтый 30-60%, красный <30%
- Числовое значение: «2 / 60 шт»

В визуальной схеме стеллажей — тултип при наведении дополнить:
- Было: название детали + количество
- Стало: название + текущий остаток + максимум + % заполненности

## Валидация на бэкенде (/prikhod POST)

```python
# Проверить что приход не превысит максимум ячейки
current_stock = get_current_quantity(part_id)
max_qty = get_max_quantity(part_id)
if current_stock + quantity > max_qty:
    flash(f'Ошибка: превышение ёмкости ячейки. Максимум {max_qty} шт, сейчас {current_stock} шт, можно добавить {max_qty - current_stock} шт')
    return redirect(url_for('prikhod'))
```

## Инициализация максимумов в БД

При запуске migrate или отдельным скриптом:
```sql
UPDATE stock SET max_quantity = (
    SELECT monthly_plan FROM parts WHERE parts.id = stock.part_id
)
WHERE max_quantity IS NULL;
```
