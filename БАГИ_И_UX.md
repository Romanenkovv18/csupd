## BUG-04 | Кнопка «Применить» в журнале срабатывает со второго клика

**Где:** /journal, JS фильтров
**Что сделать:** Найти обработчик кнопки «Применить». Проблема скорее всего в том что форма отправляется через JS но также есть дефолтный submit. Исправить:
```javascript
filterForm.addEventListener('submit', function(e) {
    e.preventDefault();
    // собрать параметры и перейти по URL
    const params = new URLSearchParams(new FormData(this));
    window.location.href = '/journal?' + params.toString();
});
```
Убедиться что нет двойных обработчиков.

## FEAT-01 | Кнопка «+ Приход» на главной с постприходным сценарием

**Где:** дашборд (/), кладовщик и выше
**Что сделать:**
- Добавить крупную кнопку «+ Приход» на главной странице (отдельный заметный блок, не мелкая кнопка в шапке таблицы)
- После успешного прихода — не просто flash-сообщение, а отдельный экран «Что дальше?» с тремя действиями:
  1. **Разместить в ячейку** — показывает код ячейки крупно (С1-П1-Я1) и название детали
  2. **Распечатать QR** — кнопка генерирует и открывает страницу печати с QR-кодом ячейки
  3. **Завершить операцию** — возврат на дашборд

**Маршрут:** GET /prikhod/success?part_id=X&quantity=Y — промежуточная страница после POST /prikhod

**QR-код:** генерировать через библиотеку qrcode (pip install qrcode[pil]). QR должен кодировать строку вида: `ЦСУПД:ЯЧЕЙКА:С1-П1-Я1:ДЕТАЛЬ:0080004096`. Страница печати — минималистичная, только QR + подпись, открывается в новой вкладке через window.print().

## FEAT-02 | Расширенный поиск деталей

**Где:** /raskhod, /prikhod, /details — все места где есть поиск деталей
**Что сделать:** Единый компонент поиска деталей с поддержкой:

1. **По артикулу** — уже есть, оставить
2. **По названию или части названия** — LIKE '%запрос%' в SQL (уже частично есть, убедиться что работает)
3. **По QR-коду** — поле ввода + кнопка «Сканировать». При вводе строки вида `ЦСУПД:ЯЧЕЙКА:С1-П1-Я1:ДЕТАЛЬ:0080004096` — автоматически парсить артикул и выбирать деталь

## FEAT-03 | Защита от ошибок и аудит изменений

### 3.1 Нельзя списать больше остатка
**Уже реализовано частично** — убедиться что работает везде: в /raskhod и в /assembly/complete. На фронте — отключать кнопку подтверждения если quantity > stock. На бэке — возвращать 400 с понятным сообщением.

### 3.2 Подтверждение крупных списаний
**Где:** /raskhod POST
**Что сделать:** Если списывается более 50% от месячного плана за раз — показывать дополнительный экран подтверждения:
```
«Вы списываете 45 шт Шайба контровочная (это 75% месячного плана).
Подтвердить?»
[Отмена] [Да, списать]
```
Порог: quantity > monthly_plan * 0.5

### 3.3 Журнал всех изменений (audit log)
Добавить таблицу audit_log:
```sql
CREATE TABLE audit_log (
    id INTEGER PRIMARY KEY,
    table_name TEXT,           -- 'stock', 'parts', 'users' и т.д.
    record_id INTEGER,         -- id изменённой записи
    action TEXT,               -- 'INSERT', 'UPDATE', 'DELETE'
    old_value TEXT,            -- JSON старого значения
    new_value TEXT,            -- JSON нового значения
    changed_by TEXT,           -- логин пользователя
    changed_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    ip_address TEXT            -- IP для безопасности
);
```
Записывать в audit_log при каждом: приходе, расходе, изменении порогов, изменении пользователей, отмене операции.

### 3.4 Отмена операции только администратором
**Где:** /journal
**Что сделать:** Добавить кнопку «Отменить операцию» в журнале — видна только admin и master. При нажатии:
- Показать модал с деталями операции
- Потребовать ввести причину отмены (обязательное поле)
- После подтверждения: вернуть детали на склад (если расход) или снять с остатка (если приход)
- Записать в audit_log и в movements с типом «отмена»
- Пометить исходную запись как отменённую (добавить поле is_cancelled в movements)

### 3.5 Фиксация кто и когда изменил запись
**Где:** таблица parts, stock
Добавить поля:
```sql
ALTER TABLE parts ADD COLUMN updated_by TEXT;
ALTER TABLE parts ADD COLUMN updated_at DATETIME;
ALTER TABLE stock ADD COLUMN updated_by TEXT;
ALTER TABLE stock ADD COLUMN updated_at DATETIME;
```
Заполнять при каждом UPDATE из сессии: updated_by = current_user.username, updated_at = datetime.now()

В интерфейсе — показывать в карточке детали: «Последнее изменение: Белоусов А. — 12.06.2026 18:02»
